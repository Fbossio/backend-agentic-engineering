# Protocolo de chats del curso

## Convención de nombres

- Gobierno: `00 - Diseño y gobierno del curso`
- Módulos: `NN - Nombre oficial del módulo`

El número utiliza dos dígitos:

- `01`
- `02`
- `03`

## Selección del siguiente módulo

El siguiente módulo será la primera fila de `docs/PROGRESO.md`
cuyo estado no sea `Completado`.

El nombre oficial se obtiene de `docs/CURSO_MASTER.md`.

## Apertura

1. Consultar GitHub, rama `main`.
2. Leer `PROGRESO.md`.
3. Leer el apartado correspondiente en `CURSO_MASTER.md`.
4. Leer decisiones relevantes de `DECISIONES.md`.
5. Leer `PLANTILLA_MODULO.md`.
6. Comunicar el nombre exacto del chat.
7. Esperar que el usuario confirme el cambio de nombre.

## Desarrollo

El módulo debe seguir la plantilla oficial y producir contenido
didáctico, demostración, laboratorio, ejercicio y evaluación.

No es obligatorio producir todo en una sola respuesta. El contenido
puede construirse por secciones, manteniendo un índice de avance.

## Cierre

El chat debe entregar:

- contenido completado;
- resumen de continuidad;
- decisiones tomadas;
- pendientes;
- archivos por crear o actualizar;
- actualización propuesta de `PROGRESO.md`;
- nombre del siguiente chat.

## Criterio de finalización

El módulo no se considera completado hasta que el contenido y el estado
actualizado estén incorporados en la rama `main`.
